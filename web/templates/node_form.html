Function reference

{{template "help" "The building blocks you can type into the Command field above. Click that field to pick from this list directly."}}

This covers the commands most nodes use. app_rpt supports further advanced functions (playing sound files, reading hardware sensors, running saved macros, and more) — check the AllStarLink/app_rpt documentation for anything not listed here, and for the full set of "control operator" (cop) sub-commands, which vary by what your node needs.

CommandWhat it does
ilink,1Disconnect the most recently connected node
ilink,2Connect to a node, listen-only — you can hear them, they can't hear you
ilink,3Connect to a node, both directions — stays connected until manually disconnected
ilink,5Announce which nodes are currently connected
ilink,6Disconnect every connected node at once
ilink,8Connect to a node, listen-only, temporary — drops automatically when the repeater goes idle
status,1Announce this node's own number and status
autopatchupStart an autopatch call (dial out through a connected phone line/gateway, if one is set up)
autopatchdnEnd the current autopatch call
cop,<n>"Control operator" — local system actions like reloading configuration. Which number does what depends on your setup; see the documentation.
macro,<n>Runs a saved sequence of commands in one dial — macros are defined elsewhere in rpt.conf
